# Rebuild and Redeploy Guide

This document outlines the process for triggering a fresh rebuild and redeployment of the Beemedha application without making functional changes to the codebase.

## Purpose

Sometimes a fresh build and deployment is needed to:
- Verify the build pipeline is working correctly
- Deploy after infrastructure changes
- Create a new draft version for testing
- Resolve deployment-related issues
